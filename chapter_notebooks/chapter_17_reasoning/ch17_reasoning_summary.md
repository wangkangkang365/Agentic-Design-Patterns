# Ch17: Reasoning（推理）总结

## 核心思想

推理增强让 LLM 从「直接回答」变为「深度思考」，显著提高复杂问题的准确率。

## 三种推理模式

### 1. Chain-of-Thought（CoT）分步推理
- **核心**：在 prompt 中要求 LLM「一步一步思考」
- **优点**：简单、无需额外调用
- **缺点**：不保证正确
- **适用**：数学题、逻辑题、多步推理

### 2. Self-Correction 自我修正
- **核心**：回答 → 检查 → 修正（可多轮）
- **优点**：能发现并修正错误
- **缺点**：多次调用，成本高
- **适用**：高风险决策、需要高准确率的场景

### 3. Plan-and-Execute 先规划再执行
- **核心**：先生成计划 → 逐步执行 → 汇总结果
- **优点**：结构化、可追踪、减少返工
- **缺点**：计划可能不合理
- **适用**：复杂多步任务

## 代码结构

### 第1步：导入依赖
- `langchain_openai.ChatOpenAI` + MiMo 模型

### 第2步：CoT 对比测试
- 同一问题，分别用「直接回答」和「CoT 分步推理」
- 对比两种方法的输出质量

### 第3步：Self-Correction 实现
- `self_correct()` 函数：初始回答 → 检查轮次 → 修正
- 支持多轮修正（max_rounds 参数）
- 自动检测「回答正确」时停止

### 第4步：Plan-and-Execute 实现
- `plan_and_execute()` 函数：生成计划 → 逐步执行 → 汇总
- 每步执行时携带上下文（之前步骤的结果）
- 最终汇总所有步骤的结果

## 重难点

### 重点
- **CoT prompt 设计**：「一步一步思考」是最简单有效的推理增强
- **Self-Correction 停止条件**：如何判断「已经正确」
- **Plan-and-Execute 上下文传递**：每步执行时如何利用之前的结果

### 难点
- **CoT 不保证正确**：模型可能在推理链中出错
- **Self-Correction 成本**：每次修正都需要额外的 LLM 调用
- **计划质量**：LLM 生成的计划可能不合理，需要兜底

## 进阶方向

- **Tree-of-Thought（ToT）**：多条推理路径并行探索
- **Graph-of-Thought（GoT）**：图结构推理，支持回溯
- **Self-Consistency**：多次 CoT，取多数投票结果
- **结合搜索**：推理 + 联网搜索（DeepSearch）
